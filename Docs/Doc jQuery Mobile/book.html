Introduction To jQuery Mobile

The mobile devices have nowadays a more and more important position to the public.

As web developers, we have to take mobile devices into account by making websites adapted to their small screens and touch interfaces.

jQuery Mobile is a very powerful framework which will help up in this task.

In this chapter, we are going to discuss why using jQuery Mobile will permit us to quickly develop a mobile application.

We are also going to discuss about how to start with this library (discussing about where to get the library, how to be "ready" to use it in your web pages and where to look for useful complementary learning resources)

What Is jQuery Mobile?

  • jQuery Mobile is a JavaScript library that simplifies the way we build a mobile website by:

    • automatically splitting our HTML page into severals pages linked between each other;

    • transforming forms elements into big and nice touch elements;

    • handling touch events;

    • supporting a lot of mobile devices;

  • jQuery Mobile is:

    • Free!;

    • Open-source and available under both MIT and GPL licences;

    • Only 20Kb;

Note
 You have to understand that the goal of jQuery Mobile is only to adapt a website for mobile devices. It just updates the HTML structure and its design, nothing else… but that’s a lot!

Warning

Some browsers do not handle “self-closing” <script> tags correctly. For example, some browsers fail to load the script if you use the following syntax:

<script src="http://code.jquery.com/jquery.min.js"></script>

Page Transitions

If you have tested the links navigation, you have probably seen that a slide transition is automatically done. You can change the type of transition by setting a data-transition="X" attribute to the corresponding <a> tag, where X is one of these following:

  • slide

  • slideup

  • slidedown

  • pop

  • fade

  • flip

Furthermore, by adding the data-rel="back" to the <a> tag, the transition will be reversed.

Warning

The flip transition uses some advanced CSS3 features and it is not supported by all platforms!

In Brief:

  1. data-transition attribute provides a lot of transition effects.

  2. data-transition has to be set on an <a> tag.

  3. We can reverse a transition thanks to data-rel="back" attribute.

Hashes And History

  • jQuery Mobile navigation is based on changes and updates to the hash location.

  • Whenever the page is changing, the hash is updated, and conversely.

  • Behind the scenes, the library updates the native history object.

Thanks to these features and despite the special management of pages in jQuery Mobile, all is transparent for the user!

Forcing The Change Of Page

We can also force the user to change page with $.mobile.changePage() method.

The first parameter is required and defines the target page to show: it can be an absolute or relative URL (as a string) or a jQuery object (like $('#secondPage')).

The second parameter is optional: it’s an object containing several properties:

  • transition: the transition to apply during the changing.

  • reverse: define the direction of the transition. It’s the same effect than data-rel="back" for links.

  • changeHash: define if the hash in the location bar has to be updated or not.

  • role: data-role value to be used when displaying the page.

  • pageContainer: the element that will contain the page to show.

  • type: "get" or "post".

  • data: an object containing data to send to the page by the Ajax request.

  • reloadPage: define if the library has to force the reload of the page (to prevent caching).

Note
 The last three properties are useful only when the page to go is an URL.
